# EasyFlowHub Roadmap

## v1.0.0

### 当前目标
- 稳定管理中心与快速笔记的 Markdown 编辑体验，避免标题、todo、图片和快捷键行为分叉。
- 修复快速笔记、多窗口和开机自启的关键使用问题。
- 收敛管理面板的公共骨架，降低新增面板和设置区块的实现成本。

### 2026-04-05 进展
- 已修复管理面板中“正文首行被误持久化为标题”的问题，标题改为只保存显式输入值，列表展示仍可从正文推导。
- 已统一快速笔记与管理中心的 Markdown 自动补全逻辑，支持无序列表、有序列表、checkbox 连续输入与括号补全。
- 已将全局 todo 解析扩展为支持有序列表 checkbox，并支持在全局 todo 面板内直接编辑文本。
- 已让全局 todo 与悬浮 todo 卡片按来源笔记分组显示，使用分割线区分不同文件。
- 已修正图片插入后的链接生成方式，改为使用 Tauri `convertFileSrc` 指向真实落盘路径。
- 已将快速笔记中的图片切换为独立附件渲染，编辑区不再直接暴露原始 markdown 图片语法，保存时再合并回文档。
- 已补上快速笔记图片的直接删除能力，并将图片区改为不放大小图、可滚动查看的独立卡片。
- 已将快速笔记回退到稳定布局，图片展示改为顶部缩略条，优先恢复窗口可见性、可编辑性和可关闭性。
- 已将快速笔记图片拖拽切回当前窗口级事件监听，避免跨窗口串写，并将关闭动作切到当前窗口强制销毁路径。
- 已为本地路径图片插入补充后端复制命令，并放开 asset protocol 对应用数据目录的访问范围。
- 已将自启动切换为显式 `--autostart` 后台启动参数，并在启动阶段保持管理窗口隐藏。
- 已抽取管理面板公共头部和设置区块组件，开始减少新面板的重复模板代码。
- 已将 todo 悬浮面板重做为极简浮窗，补上关闭、固定、刷新、新增与内联编辑能力。
- 已为 todo 悬浮面板补上来源笔记标题与回跳入口，可直接唤起管理中心并定位到原笔记。

### 待继续验证
- 需要在 Windows 打包产物中回归验证开机自启动行为，确认不会再出现终端窗口或卡住的问题。
- 需要对快速笔记图片的大图展示、跨窗口切换和历史旧笔记兼容性做一轮人工回归。
- 需要继续验证快速笔记在 `Esc`、`Alt+F4` 和工具按钮下都只关闭当前窗口，不会再出现关闭卡死。
- 需要继续回归快速笔记的图片插入与删除细节，在稳定后再恢复更强的图片编辑体验。
- 需要继续评估是否将 todo 新建入口进一步下放到管理中心聚合面板。
